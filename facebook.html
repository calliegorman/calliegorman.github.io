Redesign Challenge: Sign Up Process

Facebook’s sign up process provides a less than desirable UI interface that lacks several notable best practices that could make an easier, more pleasant user experience.

The Challenge

Research and design a new flow for signing up as a new user on Facebook within 24 hours.

The Solution

A new sign up flow for both desktop and mobile, incorporating proven best practices for sign up forms to ensure a delightful experience for the user.

Examples of current mobile designs

The Situation

The current Facebook sign up process design has several areas of improvement. I signed up for a new account for both mobile and desktop and found the experience was outdated and not user friendly.

Because I was only given 24 hours and did not have access to user interaction data from Facebook, I resorted to diving deep into sign up process best practices and comparing them with Facebook’s designs.

Review sign up process best practices

I reviewed a few reports online discussing proven best practices for sign up forms. The most prevalent items found are below.

1. Make all fields required (but limit them)

2. Show password requirements up front

3. Ask for info in steps (and make # of steps clear)

4. Auto focus the input field the user is on

5. List input title above input box so users don’t forget requirements

6. Include microinteractions for active and completed tasks

Understand current design & identify improvement areas

While reviewing the current Facebook sign up process, it was clear that they did not meet all of the requirements laid out by design gurus. The items I felt were most important to improve in a short amount of time are below.

  • Outdated UI (especially for desktop)
  • Number of form steps missing
  • No microinteractions
  • No option to view password
  • Required fields not indicated
  • No strong reason to include gender (users can change it once getting into the account)
  • No directions on what to do next after sign up is completed

Branding & High Fidelity Mockups

After reviewing the Facebook brand guidelines to ensure I used the correct Facebook blue and typeface, I applied these details into high fidelity mockups in Sketch.

Significant improvements to note are:

  • Mobile version broken up into few steps and the steps are indicated to the user
  • Password requirements, strength, and ability to view password was added
  • Microinteractions added to indicate to a user that a task is active or complete
  • Overall UI was updated to a cleaner, more professional feel

Testing

Given the time constraint of 24 hours, I was able to run a couple of informal usability tests on the prototype.

Two changes that were made to the final prototypes were:

  • Increased line height in the password strength/description section for better readability
  • Buttons highlighted to full opacity only after the input field was completed correctly

What I learned

Overall, Facebook's current design does not follow the common trends for sign up processes that have been proven to provide users with exceptional user experiences. Things like adding microinteractions when a password strength is high, highlighting the text box the user is completing, and providing a progress bar are all simple changes that provide users a better overall experience.

This was an incredible challenge to face as a new designer. The time constraint forced me to lay out a plan, work on my process, and execute as many crucial research items as I could quickly. If given more time, I would have done several additional in person, formal usability tests in addition to preference tests to ensure the design was the best it could be.
